# Contributing to Kernolog

Thank you for your interest in contributing to Kernolog! We welcome contributions of all kinds: code, documentation, bug reports, feature suggestions, and more. To keep things smooth and organized, please follow the steps below.

## How to Contribute

### 1. Fork the Repository

1. Click the "Fork" button at the top right of the repository page.
2. Clone your fork to your local machine:

```bash
git clone https://github.com/<your-username>/Kernolog.git
cd Kernolog
```

### 2. Create a Branch

Always create a new branch for your work:

```bash
git checkout -b feature/my-feature
```

Replace `my-feature` with a descriptive name.

### 3. Make Changes

* Follow the existing coding style.
* Keep commits small and focused.
* Include clear commit messages.

### 4. Test Your Changes

Ensure your changes work as expected and do not break existing functionality. Add tests if applicable.

### 5. Submit a Pull Request (PR)

1. Push your branch to your fork:

```bash
git push origin feature/my-feature
```

2. Open a pull request against the `main` branch of the original repository.
3. Fill in the PR template with a clear description of your changes.

### 6. Participate in Code Review

* Respond to feedback constructively.
* Make requested changes and update your PR.

## Reporting Issues

If you find a bug or have a feature request:

1. Go to the [Issues](https://github.com/Tonystank2/Kernolog/issues) tab.
2. Click "New Issue" and provide detailed information.

## Code of Conduct

By participating, you agree to follow the [Code of Conduct](https://github.com/Tonystank2/Kernolog/blob/main/CODE_OF_CONDUCT.md). Be respectful and supportive.

## Ways to Contribute

* **Code contributions:** Fix bugs, add features, optimize performance.
* **Documentation:** Improve README, write tutorials, add examples.
* **Testing:** Report bugs, suggest improvements, write unit tests.
* **Ideas & Feedback:** Suggest new features or improvements.

Thank you for helping make Kernolog better!
